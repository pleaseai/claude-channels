# Slack Plugin Setup

This guide walks you through setting up the Slack channel plugin from scratch â€” creating a Slack app, configuring tokens, and connecting it to Claude Code.

## Prerequisites

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI installed
- Admin access to a Slack workspace

## 1. Create a Slack App

1. Go to [api.slack.com/apps](https://api.slack.com/apps) and click **Create New App**
2. Choose **From scratch**
3. Name your app (e.g., `Claude Code`) and select your workspace

### Enable Socket Mode

1. Navigate to **Settings â†’ Socket Mode**
2. Toggle **Enable Socket Mode** on
3. Create an app-level token with the `connections:write` scope
4. Copy the token â€” this is your `CLAUDE_SLACK_APP_TOKEN` (starts with `xapp-`)

### Configure Bot Token Scopes

Navigate to **Features â†’ OAuth & Permissions** and add these **Bot Token Scopes**.

**Common (always required):**

| Scope | Purpose |
|-------|---------|
| `chat:write` | Send messages |
| `app_mentions:read` | Detect @mentions of the bot |
| `users:read` | Look up user info |
| `reactions:write` | Add emoji reactions |
| `files:read` | Download shared files |
| `files:write` | Upload file attachments |

**Channel mode** (add these if using a channel):

| Scope | Purpose |
|-------|---------|
| `channels:history` | Read messages in public channels |
| `channels:read` | View basic channel info |
| `groups:history` | Read messages in private channels (if private) |
| `groups:read` | Access private channels (if private) |

**DM mode** (add these if using direct messages):

| Scope | Purpose |
|-------|---------|
| `im:history` | Read DM history |
| `im:read` | List DM conversations |
| `im:write` | Open DM conversations |

### Enable Event Subscriptions

Navigate to **Features â†’ Event Subscriptions**:

1. Toggle **Enable Events** on
2. Under **Subscribe to bot events**, add:
   - `app_mention` â€” @mentions of the bot
   - **Channel mode**: `message.channels` (public) and/or `message.groups` (private)
   - **DM mode**: `message.im`
3. Click **Save Changes**

### App Home (DM mode only)

If using DM mode:

1. Navigate to **Features â†’ App Home â†’ Show Tabs**
2. Enable **Messages Tab**
3. Check **"Allow users to send Slash commands and messages from the messages tab"**

### Install to Workspace

1. Navigate to **Settings â†’ Install App**
2. Click **Install to Workspace** and authorize
3. Copy the **Bot User OAuth Token** â€” this is your `CLAUDE_SLACK_BOT_TOKEN` (starts with `xoxb-`)

::alert{type="warning"}
If you add scopes after installing, you must go back to **Install App** and click **Reinstall to Workspace** for the changes to take effect.
::

## 2. Install the Plugin

### Option A: Marketplace (recommended)

```bash
claude plugin add claude-channels/slack
```

### Option B: Development mode

```bash
git clone https://github.com/pleaseai/claude-channels.git
cd claude-channels
bun install
claude --dangerously-load-development-channels plugin:slack@claude-channels
```

## 3. Configure Credentials

Use the built-in configuration command:

```bash
/slack:configure <xoxb-bot-token> <xapp-app-token>
```

Or manually create `~/.claude/channels/slack/.env`:

**Channel mode:**

```env
CLAUDE_SLACK_BOT_TOKEN=xoxb-your-bot-token
CLAUDE_SLACK_APP_TOKEN=xapp-your-app-token
CLAUDE_SLACK_CHANNEL_ID=C0123456789
```

**DM mode:**

```env
CLAUDE_SLACK_BOT_TOKEN=xoxb-your-bot-token
CLAUDE_SLACK_APP_TOKEN=xapp-your-app-token
CLAUDE_SLACK_DM_USER_ID=UYourSlackUserID
```

::alert{type="warning"}
Keep this file private. It contains secrets that grant access to your Slack workspace. The file is stored outside the project directory to prevent accidental commits.
::

### Find Your Channel ID (channel mode)

1. Open Slack and navigate to the channel where the bot should operate
2. Click the channel name at the top to open channel details
3. At the bottom of the details panel, copy the **Channel ID** (starts with `C`)

Then invite the bot to the channel: `/invite @YourBotName`

### Find Your User ID (DM mode)

1. Click your **profile picture** in Slack
2. Click **Profile**
3. Click **â‹®** (more actions) â†’ **Copy member ID**

## 4. Verify the Connection

Start a new Claude Code session (or run `/reload-plugins`). You should see in the output:

**Channel mode:**
```
slack channel: connected as your-bot-name (U...)
slack channel: bound to thread ... in C...
slack channel: Socket Mode connected
```

**DM mode:**
```
slack channel: connected as your-bot-name (U...)
slack channel: DM mode â€” resolved channel D... for user U...
slack channel: bound to thread ... in D...
slack channel: Socket Mode connected
```

A new thread will be created in the configured channel or DM with the message "Claude Code session started". All messages for this session are scoped to that thread.
