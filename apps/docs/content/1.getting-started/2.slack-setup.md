# Slack Plugin Setup

This guide walks you through setting up the Slack channel plugin from scratch — creating a Slack app, configuring tokens, and registering the MCP server with Claude Code.

## Prerequisites

- [Bun](https://bun.sh) 1.3.10 or later
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) CLI installed
- Admin access to a Slack workspace

## 1. Create a Slack App

1. Go to [api.slack.com/apps](https://api.slack.com/apps) and click **Create New App**
2. Choose **From scratch**
3. Name your app (e.g., `Claude Code`) and select your workspace

### Enable Socket Mode

1. Navigate to **Settings → Socket Mode**
2. Toggle **Enable Socket Mode** on
3. Create an app-level token with the `connections:write` scope
4. Copy the token — this is your `SLACK_APP_TOKEN` (starts with `xapp-`)

### Configure Bot Token Scopes

Navigate to **Features → OAuth & Permissions** and add these **Bot Token Scopes**:

| Scope | Purpose |
|-------|---------|
| `channels:history` | Read messages in public channels |
| `channels:read` | View basic channel info |
| `chat:write` | Send messages |
| `files:read` | Download shared files |
| `files:write` | Upload file attachments |
| `reactions:write` | Add emoji reactions |
| `app_mentions:read` | Detect @mentions of the bot |

### Enable Event Subscriptions

Navigate to **Features → Event Subscriptions**:

1. Toggle **Enable Events** on
2. Under **Subscribe to bot events**, add:
   - `message.channels` — messages in public channels
   - `app_mention` — @mentions of the bot
3. Click **Save Changes**

### Install to Workspace

1. Navigate to **Settings → Install App**
2. Click **Install to Workspace** and authorize
3. Copy the **Bot User OAuth Token** — this is your `SLACK_BOT_TOKEN` (starts with `xoxb-`)

### Find Your Channel ID

1. Open Slack and navigate to the channel where the bot should operate
2. Click the channel name at the top to open channel details
3. At the bottom of the details panel, copy the **Channel ID** (starts with `C`)

## 2. Configure Credentials

Create the state directory and credentials file:

```bash
mkdir -p ~/.claude/channels/slack
```

Write your tokens to `~/.claude/channels/slack/.env`:

```env
SLACK_BOT_TOKEN=xoxb-your-bot-token
SLACK_APP_TOKEN=xapp-your-app-token
SLACK_CHANNEL_ID=C0123456789
```

::alert{type="warning"}
Keep this file private. It contains secrets that grant access to your Slack workspace. The file is stored outside the project directory to prevent accidental commits.
::

## 3. Invite the Bot to the Channel

In Slack, go to the channel you configured and invite the bot:

```
/invite @Claude Code
```

The bot must be a member of the channel to read messages and post replies.

## 4. Register the MCP Server

Add the plugin to your Claude Code MCP settings. You can configure it globally or per-project.

### Global (all projects)

Add to `~/.claude/settings.json`:

```json
{
  "mcpServers": {
    "slack-channel": {
      "command": "bun",
      "args": ["run", "/absolute/path/to/claude-channels/plugins/slack/server.ts"]
    }
  }
}
```

### Per-project

Add to your project's `.mcp.json`:

```json
{
  "mcpServers": {
    "slack-channel": {
      "command": "bun",
      "args": ["run", "/absolute/path/to/claude-channels/plugins/slack/server.ts"]
    }
  }
}
```

::alert{type="info"}
Replace `/absolute/path/to/claude-channels` with the actual path where you cloned the repository. If you move the project later, update this path accordingly.
::

## 5. Verify the Connection

Start a new Claude Code session. You should see in the startup output:

```
slack channel: connected as your-bot-name (U...)
slack channel: bound to thread ... in C...
slack channel: Socket Mode connected
```

A new thread will be created in the configured Slack channel with the message "Claude Code session started". All messages for this session are scoped to that thread.
