# Usage Guide

Once the Slack plugin is running, each Claude Code session creates a dedicated thread in your configured Slack channel or DM. All communication happens within that thread.

## Sending Messages

Simply type a message in the session thread. The plugin forwards it to Claude Code, and Claude's responses appear as replies in the same thread.

```
You (in Slack thread): Can you explain how the auth middleware works?
Claude (replies in thread): The auth middleware validates JWT tokens...
```

### Mentions

You can @mention the bot within the session thread to interact. Mentions outside the bound thread are silently dropped — only messages inside the session thread are processed.

## File Attachments

### Sending files to Claude

Attach files to your Slack message in the thread. Claude receives metadata about the attachments and can download them on demand using the `download_attachment` tool.

The plugin supports files up to **25 MB** each. Attachments are downloaded to `~/.claude/channels/slack/inbox/` with a timestamped filename.

### Receiving files from Claude

When Claude needs to share a file, it uses the `reply` tool with file paths. Files are uploaded to the Slack thread as attachments.

Up to **10 files** can be attached per message.

## Available Tools

The plugin exposes these MCP tools to Claude:

### `reply`

Send a message (and optional file attachments) to the bound thread.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | yes | Message text |
| `files` | string[] | no | Absolute file paths to attach (max 10, 25 MB each) |

Long messages are automatically split into chunks at a 4,000-character limit per chunk.

### `react`

Add an emoji reaction to a message in the thread.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message_id` | string | yes | Message timestamp (ts) to react to |
| `emoji` | string | yes | Emoji name without colons (e.g., `thumbsup`) |

### `edit_message`

Edit a message the bot previously sent.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message_id` | string | yes | Message timestamp (ts) to edit |
| `text` | string | yes | New message text |

### `download_attachment`

Download file attachments from a message in the thread.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message_id` | string | yes | Message timestamp (ts) to download files from |

Returns the local file paths where attachments were saved, ready for Claude to read.

### `fetch_messages`

Fetch recent message history from the bound thread.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `limit` | number | no | Max messages to fetch (default: 20, max: 100) |

Returns messages in oldest-first order with timestamps.

## Thread-Bound Sessions

Each Claude Code session is bound to a single Slack thread:

- **One thread per session** — Starting a new Claude Code session creates a new thread
- **Isolated communication** — Messages from other threads or channels are ignored
- **All tools operate within the thread** — `react`, `edit_message`, and `download_attachment` verify that the target message belongs to the bound thread

::alert{type="info"}
When a session ends, the thread remains in Slack as a conversation record. Starting a new session creates a fresh thread.
::

## Troubleshooting

### No messages received

- **Channel mode**: Verify the bot is a member of the channel (`/invite @BotName`)
- Check that `CLAUDE_SLACK_CHANNEL_ID` or `CLAUDE_SLACK_DM_USER_ID` is set correctly
- Ensure you're replying in the correct thread (the one created at session start)
- Check Claude Code's stderr output for connection errors

### Authentication errors

- Confirm `CLAUDE_SLACK_BOT_TOKEN` and `CLAUDE_SLACK_APP_TOKEN` are correct in `~/.claude/channels/slack/.env`
- Verify the bot token has all required scopes (see [Slack Setup](/getting-started/slack-setup#configure-bot-token-scopes))
- Re-install the app to the workspace if scopes were added after installation

### File upload failures

- Check that the file size is under 25 MB
- Verify the bot has the `files:write` scope
- Ensure the file path is absolute and the file exists
